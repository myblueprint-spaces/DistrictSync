## Problem
IA-9 is the **cross-surface humanization checkpoint** of the Flet rebuild. The mission (0013): a non-technical district admin opens the cockpit 2–3×/yr and must **trust the roster synced** — every admin-facing string humanized, verdict-first, *never* a raw `sis_type` / filesystem path / filename / raw ISO timestamp / `str(e)` exception / stack trace / raw config id. Each prior slice already enforced this locally (category-only faults, `friendly_district_name`/`friendly_timestamp`, PII-free frozen summaries, no raw `error` interpolation — each has a privacy test). IA-9's job is to **verify that invariant holds across ALL surfaces at once**, fix any straggler, **de-duplicate** the humanization helpers that have been copy-authored slice-by-slice, and lock the invariant with a **cross-surface regression test** so a future slice can't regress it. Plus persist the durable narrative product spec (`docs/claugentic-PRODUCT.md`) the whole rebuild has been serving.

**The audit was performed as part of this plan** (grep + read across every `src/ui_flet/` module). The honest result: **the pure modules are clean** (every leak surface routes through a total humanizer or names faults by category; each has a privacy test) — **BUT** there are (a) a small number of **genuine raw-exception leaks in the Setup view's error cards**, and (b) **duplicated humanization helpers** (`_anomaly_detail` ×3, `_pluralize` ×2) copy-authored across the pure modules. So IA-9 is NOT "manufacture work on a clean surface" — it is a real, bounded consolidation + a real (small) leak fix + the regression test + the doc.

## Approach
Three small, cohesive changes + one doc, all presentation-only:

1. **Consolidate humanization helpers into `humanize.py`** (the established home for pure trust-copy). Add public `pluralize(word, count) -> str` and `friendly_anomaly_detail(count, *, tense: AnomalyTense) -> str` (parametrized so each surface keeps its intended voice — see Design-Q 2). Migrate `home_status`, `run_history`, `convert_result` to import them; delete the local copies. A rename/wording change now happens in ONE place (DRY; matches the IA-8a `ENTITY_LABELS` consolidation precedent).

2. **Fix the Setup leaks in the view** (`screens/setup.py`, coverage-omitted): replace the keyring `{e}` and SFTP-test `{exc}` interpolations with fixed category copy (keyring → "Couldn't save the SFTP credential on this account. Try again, or run the app as this account."; SFTP-test → a category headline + a **bounded reason** per Design-Q 1). Tighten the 2 validator `str(exc)` cards to fixed copy. All via `components.ErrorCard`/`HealthVerdictBanner` in the verdict-first voice.

3. **Cross-surface regression test** (`tests/test_ui_flet_humanization_sweep.py`, COUNTED): a parametrized "planted-sentinel" sweep over the pure derivation modules — feed each a record/config carrying a fake path `/secret/roster.csv`, a fake `sis_type` `SECRET_SIS`, a raw ISO, and an exception-shaped string, exercise every status/branch, and assert none of those substrings appears in any emitted `headline`/`detail`/`status_label`/`output_labels`. Locks the invariant as far as testable at the pure layer (the views are omitted, but the pure modules ARE the leak surface that feeds them).

4. **`docs/claugentic-PRODUCT.md`** — the durable narrative product spec (outline below).

**Alternatives rejected:** (a) *flatten `_anomaly_detail` to one string* — rejected: the per-surface tense ("in the most recent run" for Run History vs bare for Home/Convert) is intentional voice, so the shared helper is parametrized, not flattened. (b) *sanitize `test_connection` in the core* — rejected: zero-core guardrail; the view sanitizes the string it renders. (c) *a runtime view test asserting no-leak* — rejected: views are coverage-omitted and instantiating flet controls in unit tests is the documented anti-pattern; the pure sweep is the right altitude.

## `docs/claugentic-PRODUCT.md` — outline (durable narrative product spec)
A **new** harness doc (not in the 8-noise set; distinct from the `/product` AC spec). Concise, honest, matches what LANDED (IA-1..IA-8a). Structure:

- **Who it's for** — a non-technical BC school-district admin who opens DistrictSync 2–3×/yr (first-run setup; an occasional check/re-config). Not an engineer; must never be shown a machine id, path, or stack trace.

- **The promise** — a **verdict-first cockpit**: open it and immediately know *is my roster syncing?* — humanized, never a raw id/path/trace; the nightly sync runs unattended; setup is a calm one-scroll flow.
- **The design language** — verdict-first (HEALTHY/WARNING/FAILED spine), plain-language everywhere (`friendly_district_name`/`friendly_timestamp`), category-only faults, PII-free, never-crash floors (`ErrorCard`), the "your nightly sync keeps running in the background" decouple reassurance.
- **Per-surface flow + states** (matching what LANDED — one subsection each):
  - **Home** — the 3-way dashboard (onboarding if `needs_setup`; else verdict + metrics or a calm degraded/empty/stale WARNING). States: loading (sync read), empty (no runs), degraded (log unreadable), error (never-crash floor).
  - **Convert** — pick a folder → convert on a worker thread → verdict + tiles + quality; SFTP pre-flight; anomaly-ack gate; single-flight guard. States: empty (no folder), running (spinner), error (fixed category card), needs-ack.
  - **Run History** — verdict banner + read-only run table (no error column); None→unavailable, []→no-runs. States: empty, degraded, error.
  - **Setup** — one-scroll folders + schedule + SFTP; gated saves; keyring credential round-trip; calm category error cards (post-IA-9). States: validation error, permission-denied, credential-unreadable.
  - **Mapping** — review the active district config + what it produces + switch to another pre-built one (gated Apply); degraded config rendered calmly. States: degraded config, no-op/broken Apply disabled.
  - **Help** — KB link-out + support email, offline-readable. States: static (no async).
- **What "good" feels like** — calm, legible, reassuring; an admin closes the window trusting the sync; no jargon, no dead ends, no raw machine strings ever.
- **What's deliberately out (today)** — the full column-mapping editor (IA-8b, ROADMAP); bundled-docs rendering in Help (link-out only). Honest about scope.
