- **Parent program:** [`0013-flet-production-redesign.md`](0013-flet-production-redesign.md) (DS-1 row) · follows PLAT-1 (`tokens`/`theme`), PLAT-3 (`546a200`), PLAT-2 (`70587ea`)

## Problem
PLAT-1 shipped `tokens.py` + `theme.py`, but the **design-system layer** above them is missing: there's **no verdict-first spine** (the product's core is plain-language green/amber/red trust — `0013` Vision), and the visual building blocks are **re-implemented inline per surface** — `shell.py` hand-rolls its rail buttons + placeholder card, `picker_field.py` its "Browse…" button, `screens/setup.py` its buttons + cards. Two concrete debts: (1) that duplication will multiply across IA-1..IA-8; (2) `tokens.color_status_warning` is the **sky-blue accent** (`MB_ACCENT`), not amber — a "warning" verdict that doesn't read as caution and has **no contrast pair** in `UI_CONTRAST_PAIRS`. DS-1 establishes the reusable component primitives + the pure verdict-mapping spine so every later surface drops them in — and adopts them in the surfaces that exist **now**, so they ship with real consumers, not speculatively.

## Goals / Non-goals
- **Goal:** a **pure verdict module** (`src/ui_flet/verdict.py`, COUNTED, no `flet` import — follows `nav.py`): a `Verdict` enum (`HEALTHY`/`WARNING`/`FAILED`) + `verdict_visuals(v) -> VerdictVisual` mapping it to a token colour + an `ft.Icons` name (string) + a default plain-language headline + a **non-colour cue** (icon/label — verdict is never colour-alone). Trust-critical + cheaply tested.
- **Goal:** **fix + extend the verdict tokens — and DECOUPLE the theme (RC1).** `color_status_warning` becomes AA-safe **amber `#B45309` (amber-700; amber-600 `#D97706` FAILS AA — 3.19 white-on-fill — per RC2)**, used **only by the verdict layer**. BUT `theme.py:26` currently binds the M3 `secondary` role to `color_status_warning`, so repointing it would recolour `secondary` app-wide — therefore `theme.py` repoints `secondary` (and `on_secondary` if painted) to **`MB_ACCENT`** directly, keeping the theme's secondary = sky-blue **unchanged**. The warning contrast pair the banner paints — **white-on-amber-fill (`color_on_action` on `#B45309` = 5.02, clears AA)** — is added to `UI_CONTRAST_PAIRS` and gated. Brand primitives untouched; the only semantic move is warning→amber, fully decoupled from the theme.
- **Goal:** **component primitives** (`src/ui_flet/components.py`, view): `card(...)` (a `gradient=` option for the placeholder hero + a flat default — RC3), `primary_button(...)` / `secondary_button(...)` with a **`disabled_bgcolor` / disabled-state map param** so the security Save-gate styling survives (RC3), a muted/text `text_button(...)` variant (the shell Exit), `ErrorCard(...)`, `HealthVerdictBanner(verdict, headline, detail)` — all from `tokens` + `verdict_visuals`, contrast-guaranteed, light-only, per the proven 0.85.3 forms in `shell.py`.
- **Goal (the anti-speculation move):** **adopt the primitives in the existing surfaces** — refactor `shell.py` (placeholder card + rail/exit buttons), `picker_field.py` (Browse button), `screens/setup.py` (buttons/cards) to the factories. **Each surface's exact form is preserved via the factory params (RC3)** — critically `screens/setup.py`'s **stateful disabled-bgcolor Save-gate** (`setup.py:86-97`, the security affordance) keeps its disabled styling byte-for-byte; the `shell.py` gradient hero card and muted Exit `TextButton` likewise. A form that can't be cleanly factored stays inline (don't force-fit). DRY + every primitive gets a **real consumer this slice**.
- **Goal:** **`HealthVerdictBanner`/`ErrorCard` ship exercised** — though their first *live-data* consumer is IA-3, `0013`'s DS-1 row explicitly front-loads them as the verdict-first spine; this slice adds a **render-demo** (the three verdict variants + an ErrorCard) reachable behind a dev flag / a temporary placeholder route so they're visually exercised now, not dead code.
- **Non-goal:** **deriving** which verdict from real state (AppConfig/run-history/staleness) — that's **IA-3**'s pure status-derivation module; DS-1 only maps a *given* `Verdict` to visuals.
- **Non-goal:** new surfaces / wiring `HealthVerdictBanner` to live data (IA-3 Home is its first data consumer — DS-1 ships it ready + unit-mapped, not data-bound), the grouped-rail prominence (IA-1), dark mode (light-only).
- **Non-goal:** touching the ETL/CLI core. Zero `src/etl|config|sftp|scheduler|main.py` change.

## Approach
- **`verdict.py`** — pure, flet-free (`Verdict` enum + a frozen `VerdictVisual` dataclass carrying `color: str` (a token hex), `icon: str` (an `ft.Icons` member name, resolved in `components.py` like `nav.py`/`shell.py` do), `headline: str`, `tone: str`). `verdict_visuals(v)` is a total mapping over the enum (a test asserts every member maps + the colour is one of the AA-safe verdict tokens). This is the trust spine: the mapping is the single source of "what healthy/warning/failed *looks and reads* like."
- **`tokens.py`** — replace `color_status_warning = MB_ACCENT` with an AA-safe amber (e.g. amber-700 family) + add the warning fg/bg pairs to `UI_CONTRAST_PAIRS` (white-on-amber and/or amber-on-surface, whichever the banner paints). `contrast_ratio` already gates them. `MB_ACCENT` stays a primitive (sky accent for non-verdict use).
- **`components.py`** — view factories returning `ft.Control`s from tokens + `verdict_visuals`. `HealthVerdictBanner` paints the verdict colour + the **icon** (non-colour cue) + headline + optional detail. `ErrorCard` is the failed/error surface scaffold. `card`/`*_button` capture the repeated container/button shapes (radius, padding, border, text style) already in `shell.py`. Pure helpers (e.g. any text/elision logic) factored + counted; the control-assembly glue coverage-omitted.
- **Adoption refactor** — swap the inline shapes in `shell.py`/`picker_field.py`/`screens/setup.py` for the factories. Keep visuals byte-equivalent (same tokens, radii, padding) so it's a pure DRY refactor; the existing tests + manual `DISTRICTSYNC_UI=flet` confirm no regression.

## Architecture & holistic fit
- **Codebase fit** — presentation-only, tiered: `tokens` (values) → `verdict` (pure intent mapping) → `components` (token-consuming controls) → surfaces (consume components). Mirrors the existing `nav.py` pure-model precedent (flet-free, string icon names). Zero core touch → architect gate trivially met.
- **Quality dimensions:** `product-ux` (verdict-first spine; verdict never colour-alone — icon + label; AA contrast on every painted pair) · `maintainability-structure` (one component source, DRY the inline duplication, tiered tokens) · `testing` (verdict mapping + tokens contrast counted; view glue omitted) · `reliability-resilience` (ErrorCard is the reusable error-state surface the never-crash shell needs).
- **Future-proofing** — IA-3 derives `Verdict` from state → feeds `HealthVerdictBanner`; IA-1+ surfaces build from `card`/buttons/ErrorCard; the `tone`/non-colour-cue is in the model from day one so a11y isn't retrofitted.
