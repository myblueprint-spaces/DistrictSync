## Problem
The new Flet UI must let an admin pick the **input GDE folder** and **output folder** (and later sample files for mapping) — the workflow's file-picking points, today served in Streamlit by the tkinter `folder_picker.pick_directory()`. tkinter doesn't belong in the Flet app (the conventions doc replaces it with the 1.0 `ft.FilePicker` async service, dropping the tkinter hidden-import for the real UI). No Flet picker exists yet, and the Flet surfaces are placeholders. A picked path is **untrusted input to `run_pipeline`** (which validates `input_dir.exists()/is_dir()` and `sys.exit(1)`s otherwise), so the picker must validate at the boundary, not hand a raw path to the core.

## Goals / Non-goals
- **Goal:** a reusable **`ft.FilePicker` async-service wrapper** (register on `page.services` once; `await get_directory_path()` / `pick_files()`), per the conventions-doc 0.85.3 contract — the single Flet picking primitive every surface reuses.
- **Goal:** **boundary path-validation** (pure, tested) mirroring the CLI/`run_pipeline` checks — an input dir must exist + be a directory; an output dir must be creatable/writable — so no surface passes an unvalidated picked path onward.
- **Goal:** a themed, reusable **`PickerField`** component (label · "Browse…" button · chosen-path display · inline valid/invalid status), built from the existing PLAT-1 tokens.
- **Goal:** wire it into the **first real file-picking surface** — a minimal **Setup "folders" surface** replacing the `setup` placeholder: pick input + output folders, choose SIS type, validate, and **persist to `AppConfig`** (`input_dir`/`output_dir`/`sis_type` → `save()`), which flips `is_complete()` — a complete, meaningful milestone on its own.
- **Non-goal:** running the pipeline / the worker-thread `JobRunner` / verdict / write-guard — **IA-5** (the Convert surface reuses `PickerField` there).
- **Non-goal:** the scheduler / SFTP / keyring / elevation steps of Setup — **IA-4** (HIGHEST-CARE; it extends this same Setup surface with those steps).
- **Non-goal:** the Convert + Mapping surfaces — **IA-5 / IA-8** reuse the same `PickerField`/wrapper (this is how the picker reaches "everywhere": one component, dropped in per surface as each lands).
- **Non-goal:** touching the ETL/CLI core, `run_pipeline`, the tkinter `folder_picker.py` (it stays until CUT-1 removes Streamlit). Zero `src/etl|config|sftp|scheduler|main.py` change.

## Approach
- **`src/ui_flet/filepicker.py`** (new, COUNTED): the async-service wrapper + boundary validation. `async def pick_directory(page, title, initial) -> str \| None` / `async def pick_files(page, ...) -> list[str]` register `ft.FilePicker()` on `page.services` once and await the result (conventions-doc contract). **The idempotent-append (`if fp not in page.services`) stays OUTSIDE `# pragma: no cover`** and is the unit-tested behavioural contract (mocked `page.services` list); only the `await`-the-dialog glue is pragma'd (RC5). **Validation is split by purity (RC3):** `validate_input_dir(path)` is **pure** — `Path(path).resolve()` then `exists() and is_dir()` (rejects a missing path or a file-as-path; UNC/symlink accepted, documented). `validate_output_dir(path)` is **pure-structural** — parent `exists() and is_dir()` (rejects a path whose parent is a file). Writability is **NOT** claimed pure — an optional **effectful** `check_writable(path)` (`os.access` / probe) is a *separate, tmp-dir-tested* helper, and a TOCTOU note defers the real guarantee to the loader's existing backup-and-restore atomic write. Picking runs on the Flet event loop (async) → **no worker thread** (the `JobRunner` is only for the blocking `run_pipeline`, IA-5).
- **`src/ui_flet/picker_field.py`** (new, coverage-omitted view): `PickerField` — a token-themed control with a label, a "Browse…" `FiledButton` that calls the async wrapper, a path display, and an inline valid/invalid line; takes a validator + an `on_change` callback. Pure-logic (the valid/invalid + display text) factored where cheaply testable.
- **`src/ui_flet/screens/setup.py`** (new, coverage-omitted view) + a small **pure** `setup_state` helper (covered): the real Setup folders surface. Two `PickerField`s (input, output) + a **SIS/district dropdown sourced from `src/config/loader.available_configs()`** (`loader.py:113`, the existing enumerator all three Streamlit pages use — DRY) **displaying each config's `district_name`** (via `load_config(...).district_name`, as `01_Setup_Wizard.py:553-560` does), not raw ids (RC2). A "Save" affordance that persists to `AppConfig` and reflects `is_complete()`. **Block-not-flag (RC3):** Save is **structurally gated** — disabled while either path is invalid, so an invalid `input_dir`/`output_dir` can never reach `AppConfig.save()` (which would flip a *false* `is_complete()` and feed `run_pipeline` a path it `sys.exit(1)`s on). Honest in-surface note that scheduling/SFTP arrive next (IA-4) — not a dead control.
- **`src/ui_flet/shell.py`** (edit): `build_screens` swaps the `setup` placeholder factory for the real surface by binding `page` via **`functools.partial(build_setup, page)`** (RC4) — the dict's value type stays `Callable[[], ft.Control]` **unchanged**, so the other five placeholders + `render()`'s uniform `screens[id]()` call + the documented "split `shell.py` before IA-1" boundary are all **untouched**. No factory-signature change.

## Architecture & holistic fit
- **Codebase fit** — presentation-only; the wrapper calls the core's *validation shape* (not the core) and persists via `AppConfig.save()` (existing public API). Strict layering held; zero core change → architect core-untouched gate trivially met.
- **Quality dimensions:**
  - `security` — the **boundary**: a picked path is untrusted; validate (exists/is_dir/writable) before persist and before any future hand-off to `run_pipeline`. Mirror, don't bypass, the CLI's checks. No path injected into the core unvalidated.
  - `reliability-resilience` — the async picker returns `None` on cancel/no-GUI (mirrors tkinter's `None` contract); the surface degrades to manual state, never crashes. Validation is fail-soft in the UI (inline message), fail-loud at the core boundary.
  - `product-ux` — a non-technical admin gets a native "Browse…" dialog returning a real server-side path (the app runs on the district server), with plain valid/invalid feedback; picking a folder + saving is a complete, reassuring "you're set up" moment.
  - `testing` — pure validation + `setup_state` covered (80% gate holds); async glue + views coverage-omitted (`# pragma: no cover` / pyproject omit), exercised via `DISTRICTSYNC_UI=flet`.
  - `maintainability-structure` — one picker primitive + one field component reused by every later surface (DRY); the tkinter picker is not ported (deleted with Streamlit at CUT-1).
- **Future-proofing** — IA-5 (Convert) and IA-8 (Mapping) reuse `PickerField`/`pick_files`; IA-4 extends this Setup surface with scheduler/SFTP steps. The picker reaches "everywhere" via reuse, not duplication.
